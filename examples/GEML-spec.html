GEML — General Expressive Markup Language · Mini Spec (Draft)

EN — GEML is a plain-text markup language for structured, expressive documents: fully legible as plain text, expressing every kind of structured content through one typed-block primitive, with build-time reference checking, and hosting external diagram DSLs without defining a diagram language of its own.

中文 — GEML 是一种用于结构化、富表达力文档的纯文本标记语言：无需渲染即可纯文本 完整阅读，各类结构化内容统一通过单一类型块原语表达，引用在构建时校验，并托管外部图形 DSL 而自身不内建图形语言。

EN — The key words MUST, MUST NOT, MAY and SHOULD denote requirement levels: MUST / MUST NOT an absolute requirement or prohibition, SHOULD a recommendation, MAY an optional, permitted behaviour.

中文 — 关键词 MUST（必须）、MUST NOT（必须不）、MAY（可）、SHOULD（应）表示要求 等级：MUST / MUST NOT 为绝对要求或禁止，SHOULD 为推荐，MAY 为可选且被许可的行为。

1. Constraints / 约束

EN

  1. A .geml file MUST be fully readable as plain text without rendering.
  2. Code, diagrams, tables, math and callouts MUST share the single typed-block

primitive (see 3. Typed-block primitive / 类型块原语); no per-content grammar.

  1. Every block MAY carry a stable id; references MUST be resolved and validated

at build time (see 5. Inline content & links / 内联内容与链接).

  1. Graphics MUST embed an external DSL; the format defines the hosting protocol

only, never a diagram language (see 7. Graphics / 图形).

  1. There is no raw-HTML escape hatch; semantics are not tied to any backend.
  2. Headings use ATX # only. Setext headings and ---/=== thematic-break or

frontmatter rules are not part of GEML.

中文

  1. .geml 文件必须无需渲染即可作为纯文本完整阅读。
  2. 代码、图形、表格、公式、提示框必须共用唯一的类型块原语（见 3. Typed-block primitive / 类型块原语）；

不为每种内容单设语法。

  1. 每个块可携带稳定 id；引用必须在构建时解析并校验（见 5. Inline content & links / 内联内容与链接）。
  2. 图形必须内嵌外部 DSL；格式只定义托管协议，绝不内建图形语言（见 7. Graphics / 图形）。
  3. 不存在原始 HTML 逃逸口；语义不绑定任何后端。
  4. 标题只用 ATX #。setext 标题与 ---/=== 分隔线、frontmatter 规则不属于 GEML。

2. Document model / 文档模型

EN — A document is a sequence of blocks, in two shapes: flow blocks (paragraphs, headings, lists — body parsed as inline GEML) and typed blocks (fenced; body handling decided by the block type, raw or flow). Every block MAY carry an attribute object {#id .class key=val}. Inline content exists only inside flow blocks.

中文 — 一篇文档是块的序列，块只有两种形态：流式块（段落、标题、列表—— 正文按内联 GEML 解析）与类型块（带围栏；正文如何处理由块类型决定，raw 或 flow）。每个块可携带属性对象 {#id .class key=val}。内联内容只存在于流式块中。

2.1 Lists / 列表

EN — A list is a run of item lines: leading indentation, a marker (-/*, or digits + . for ordered, whose first number is the list start), a space, and the item's inline content. An item MAY begin with a task marker ([ ]/[x]/[X] + space). Nesting is by indentation (column count): an item indented more opens a nested list under the previous item; less closes back. A blank line between siblings makes the list loose, otherwise it is tight. A multi-paragraph item is not GEML — rich content goes in a typed block.

中文 — 列表是若干条目行：前导缩进、一个标记（-/*，或有序的数字加 .， 首个编号即列表 start）、一个空格、以及条目的内联内容。条目可以任务标记开头 （[ ]/[x]/[X] 加空格）。嵌套由缩进决定（按列计）：缩进更深的条目在上一条目下 开启嵌套列表，更浅则收回外层。同级条目间的空行使列表松散（loose），否则紧凑 （tight）。多段落条目不属于 GEML——丰富内容请放进类型块。

3. Typed-block primitive / 类型块原语

EN — One fence for all structured content; the type name dispatches handling.

=== <type> <attrs>?
<body>
===
  • The fence is a run of = (≥3). A block closes on a run of = of the opening

length, or — when it has an #id — on a labeled fence === #id.

  • Nesting: a longer outer fence (==== wraps ===, as above), or give each block

an #id and close it with === #id — a local close that never depends on counting = (recommended when a body itself contains fences).

  • The type registry declares each type's body mode: raw (verbatim, e.g.

code with lang=, diagram/table with format=, math), flow (parsed, e.g. note), or data (one key=val per line, e.g. meta).

  • An unknown type is a build warning; its body is preserved as raw.

中文 — 所有结构化内容共用一种围栏；类型名决定处理方式。

  • 围栏是连续的 =（≥3 个），闭合围栏长度与开围栏一致。
  • 嵌套用更长的围栏（==== 包住 ===）——如上面的代码清单所示。
  • 类型注册表声明每种类型的正文模式：raw（原样，如带 lang= 的 code、带

format= 的 diagram/table、math）、flow（解析，如 note）或 data（每行一个 key=val，如 meta）。

  • 未知类型产生构建告警，其正文按 raw 保留。
document      = { block } ;
block         = flow-block | typed-block ;

typed-block   = fence , SP , type , [ SP , attrs ] , NL , body , close-fence ;
fence         = "===" , { "=" } ;            (* open: N equals signs, N >= 3 *)
close-fence   = fence ;                       (* same length as open *)
type          = NAME ;
body          = { LINE } ;                    (* raw, flow or data per registry *)

flow-block    = heading | list | paragraph ;
heading       = "#" , { "#" } , SP , text , [ SP , attrs ] , NL ;
paragraph     = text-line , { text-line } ;

list          = item , { item | blank-line } ;
item          = indent , marker , SP , [ task ] , text , NL ;
marker        = "-" | "*" | DIGIT , { DIGIT } , "." ;
task          = "[" , ( " " | "x" | "X" ) , "]" , SP ;
indent        = { " " | TAB } ;

attrs         = "{" , { attr-item , [ SP ] } , "}" ;
attr-item     = id-attr | class-attr | kv-attr ;
id-attr       = "#" , NAME ;
class-attr    = "." , NAME ;
kv-attr       = NAME , "=" , value ;
value         = bare-word | quoted-string ;

NAME          = ALPHA , { ALPHA | DIGIT | "-" | "_" } ;

4. Attributes & identifiers / 属性与标识符

EN

  • {#budget} sets block id budget. Ids MUST be unique per document.
  • {.warning} adds a semantic class (no styling implied).
  • {caption="Annual cost"} and other key=val pairs are type-defined params.
  • A heading auto-derives an id from its text unless one is given (this section

carries an explicit {#attributes}).

  • Attribute value typing: a quoted "…" is always a string; true/false is a

boolean; a bare word matching integer/float syntax is a number; any other bare word is a string. Arrays, dates and nested tables are not supported.

  • A === meta block holds document metadata as one key=val per line, using the

value typing above.

  • Attribute order is insignificant; the recommended order is #id, then .class,

then key=val.

中文

  • {#budget} 设定块 id 为 budget，文档内 id 必须唯一。
  • {.warning} 添加语义类（不含样式）。
  • {caption="年成本"} 等 key=val 是各类型自定义的参数。
  • 标题若未显式给 id，则按文本自动生成（本节显式标了 {#attributes}）。
  • 属性值类型：带引号 "…" 恒为字符串；true/false 为布尔；匹配整数/浮点语法

的裸词为数字；其余裸词为字符串。不支持数组、日期与嵌套表。

  • === meta 块以每行一个 key=val 承载文档元数据，沿用上述属性值类型规则。
  • 属性顺序不影响语义；推荐顺序为 #id、.class、key=val。

5. Inline content & links / 内联内容与链接

5.1 Inline elements / 内联元素

EN — Inline elements appear only inside flow blocks.

  • Emphasis/strong delimiters MUST attach to a non-space character and MUST NOT

span block boundaries.

  • Block-level math uses the === math typed block, e.g. the steady state

Steady state.

  • An embed ![…] renders/plays its source in place (never navigates), while a

link […] navigates. as ∈ {image, audio, video}, inferred from the extension.

\[\frac{dy}{dt} = 0 \;\Rightarrow\; y^* = \frac{a}{k}\]

中文 — 内联元素只出现在流式块内部，例如行内公式 \(y = a\,e^{kt}\)。

5.2 Links & references / 链接与引用

EN — Internal and cross-document references are validated at build time.

  • External link options go in the attribute object: [text](url){rel=nofollow target=_blank}.
  • An unresolved #id, other.geml#id, or [^id] is a build error.
  • A footnote target MAY be written as a definition [^id]: text on its own line

(Markdown-style): it records a note block with that id, so [^id] resolves.

  • Note (non-normative): backlinks and graph views are a derived inverted index

over resolved references; GEML adds no syntax for them.

中文 — 内部与跨文档引用均在构建时校验。

  • 外链选项放进属性对象：[文字](url){rel=nofollow target=_blank}。
  • 无法解析的 #id、other.geml#id 或 [^id] 是构建错误。
  • 脚注目标可写成单独一行的定义 [^id]: 文本（Markdown 风格）：它记录一个带该 id 的

note 块，使 [^id] 得以解析。

  • 注（非规范）： 反向链接与图谱是对已解析引用的倒排索引，由工具层提供；GEML

不为此新增语法。

5.3 Recognition order & emphasis / 识别顺序与强调

EN — Inline parsing runs in two phases and gives every input exactly one parse. Phase 1 (atoms, left to right): backslash escapes, code spans, inline math; then images, links, auto-refs, footnote refs (a link/ref never nests in another). Text between atoms is literal, and an escaped delimiter is a literal atom — never emphasis. Phase 2 (emphasis) runs over each literal text run between atoms (never across an atom or a block boundary). A delimiter run is a run of *, or of two or more ~. A run may open when left-flanking and close when right-flanking: it must hug a non-space, and on a punctuation-facing side needs whitespace or punctuation beyond. One left-to-right scan pairs each closer with the nearest matching opener, under the rule of three. A * pair is emphasis (or strong when both runs have two or more); a ~~ pair is strikethrough; any unpaired delimiter is literal. This is the CommonMark algorithm restricted to * and ~~ — GEML has no _ emphasis.

中文 — 内联解析分两阶段，并为每个输入给出唯一解析。阶段一（atom，从左到右）： 反斜杠 转义、代码片段、内联数学；再图片、链接、自动引用、脚注引用（链接/引用不得相互嵌套）。 atom 之间的文本是字面文本，被转义的定界符是字面 atom，绝不参与强调。阶段二（强调） 在每段 atom 之间的字面文本上运行（不跨 atom，也不跨块边界）。定界符游程是 * 的连续串，或 两个及以上 ~ 的连续串。左侧贴合者可开、右侧贴合者可闭：须贴合非空白，且在朝向标点 的一侧需另有空白或标点。一次从左到右的扫描把每个可闭者与最近的可开者配对，并施加 rule of three。* 对是强调（两游程都有两个及以上时为加重）；~~ 对是删除线；未配对者为字面。 这是 CommonMark 算法在 * 与 ~~ 上的限定版——GEML 没有 _ 强调。

6. Tables / 表格

EN — Block type table, two interchangeable bodies parsed to one model. Form (a) is shown as source; form (b) below is a live table referenced by FY2025 revenue by segment ($M).

=== table {#cost caption="Annual cost"}
| Plan  | Months | Rate |
|-------|-------:|-----:|
| Basic |      1 |   30 |
| Pro   |      2 |   30 |
===
  • Merged cells are declared, not drawn: span="r2c1:2x1".
  • compute defines computed columns (Name = expr, ;-separated) evaluated per

row over + - * / ( ) by header name or column lettercol-letter; a formula MAY reference an earlier computed column. summary defines one foot row whose cells are literals or arithmetic over the aggregates sum, avg, min, max, count (each folding one column). A [printf] after a name (e.g. [%.1f%%]) sets numeric display. No cell/range addressing, relative-row refs, conditionals, remote(), lookups, or embedded code.

A table MAY also load its data from an external file (§6), fetched at render time with the body left empty:

Column letters are A, B, C … by position, as in a spreadsheet.

中文 — 块类型 table，两种可互换正文，解析为同一模型。下面是一个实地表格 （数据形态、带计算列），可被 FY2025 revenue by segment ($M) 引用。

  • 合并单元格用 span 声明、不画线：span="r2c1:2x1"。
  • compute 定义计算列（列名 = 表达式、; 分隔），按表头名或列字母逐行用

+ - * / ( ) 求值，可引用更靠前的计算列；summary 定义表尾一行，单元格为字面量 或对聚合 sum、avg、min、max、count（各折叠一列）的算术。列名后的 [printf] （如 [%.1f%%]）控制数字显示。不支持单元格/区域寻址、相对行引用、条件、 remote()、查表与嵌入代码。

7. Graphics / 图形

EN — Block type diagram hosts an external diagram DSL. A live diagram follows; the listing form would simply wrap it in a longer fence.

  • format selects a pluggable renderer (mermaid, graphviz, d2, plantuml, …).
  • Body is raw and passed verbatim to that renderer.
  • A processor MUST expose the renderer registry and MUST NOT interpret the body.

An unknown format is a warning; body is preserved.

  • #flow makes the diagram referenceable: see Review flow.

中文 — 块类型 diagram 托管外部图形 DSL。format 选择可插拔渲染器；正文为 raw 原样交给渲染器；处理器必须暴露渲染器注册表且不得自行解释正文；未知 format 产生 告警并保留正文；#flow 让该图可被 Review flow 引用。

8. Conformance / 一致性

EN — A conforming processor MUST:

  1. Parse the typed-block primitive (3. Typed-block primitive / 类型块原语) and the attribute object

(4. Attributes & identifiers / 属性与标识符).

  1. Build a document model in which every block id is unique and resolvable.
  2. Emit an error on any unresolved internal/cross-doc reference (5. Inline content & links / 内联内容与链接).
  3. Treat an unknown block type and an unknown diagram format as warnings,

never errors, preserving the body verbatim.

  1. Resolve inline emphasis (5. Inline content & links / 内联内容与链接) and list nesting (2.1 Lists / 列表) so that

every input has exactly one parse.

  1. NOT require any specific editor, and NOT depend on raw HTML.

A conformance suite accompanies the spec — input .geml paired with a normalized projection of the expected document model — and is the normative reference a second, independent implementation must reproduce.

中文 — 合规处理器必须：解析类型块原语（3. Typed-block primitive / 类型块原语）与属性对象 （4. Attributes & identifiers / 属性与标识符）；构建块 id 唯一且可解析的文档模型；对任何无法解析的内部/ 跨文档引用报错误（5. Inline content & links / 内联内容与链接）；把未知块 type 和未知图 format 当告警而非 错误并原样保留正文；解析内联强调（5. Inline content & links / 内联内容与链接）与列表嵌套（2.1 Lists / 列表），使每个输入 恰好一个解析；不依赖任何特定编辑器、不依赖原始 HTML。规范配套一套一致性测试集—— 输入 .geml 与期望文档模型的归一化投影成对——是第二个独立实现必须复现的规范参照。
